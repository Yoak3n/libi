# mirror

Bilibili 二游社区跨群体互动分析平台。以 [troll](../troll) 为基础数据引擎，引入多级 AI Agent 流水线，系统性地分析不同二游玩家群体在「对方社区」视频下的互动行为，量化各群体的攻击性、串门频率、拉踩倾向等跨群体特征，最终输出社区关系图谱与行为画像。

## 核心理念

> **troll** 解决「怎么拿数据」，**mirror** 解决「谁在谁的地盘上说了什么」。

**核心研究对象**：二游（二次元游戏）玩家群体——原神、崩坏：星穹铁道、绝区零、明日方舟、鸣潮、碧蓝航线、少女前线等。

**核心研究问题**：群体 A 在群体 B 的游戏相关视频评论区中，表现出的**互动行为特征**是什么？不同群体之间是否存在「攻击性差异」「串门不对称性」「拉踩倾向」等系统性的行为模式差异？

**与常规舆情分析的区别**：mirror 不关心「原神玩家如何看待原神」，而关心「原神玩家在鸣潮视频下说了什么」——即跨群体的**边界互动**。这需要 AI 同时识别：

1. 视频本身的「归属社区」（这是谁家的地盘）
2. 评论者的「出身社区」（他是哪个群体的成员）
3. 跨社区评论的「行为类型」（友善串门、恶意引战、对比拉踩、理性讨论等）

## 核心工作流

### 1. 数据采集（复用/扩展 troll）

设计目标：**覆盖多个二游社区的视频资源**，确保每个视频有明确的「归属社区」，且采集到的评论能覆盖到跨群体互动。

| 模式 | 说明 |
|------|------|
| **关键词扫描** | 以各二游名称/别称/标签为关键词（如「原神」「星穹铁道」「绝区零」「鸣潮」「明日方舟」），通过 Bilibili 搜索 API 批量获取相关视频。可以一次指定多关键词列表，系统自动将搜索结果中的视频按「归属社区」打标。 |
| **指定视频列表** | 允许用户直接提供一个 BVID/AVID 列表 + 对应的归属社区标签。适合精确圈定分析范围。 |
| **分时段分组** (可选) | 在以上两种模式基础上，按时间窗口将视频分组。例如按「周/月/季度」切分，观察跨群体互动行为在不同时间段的变化。 |

输出：结构化视频 + 评论数据，每个视频附带 `affiliated_community`（归属社区）标签，存储到 SQLite/PostgreSQL。

### 核心设计决策：一指标一 Agent

mirror 不会让一个 Agent 同时分析 7 个维度的行为特征——那会让 prompt 过于复杂、输出不稳定、难以调试。

反之，**每条跨社区评论会依次被多个高度专注的 Agent 分析**，每个 Agent 只负责一个量化指标，输出一个标准化分数。最后 Video Agent 汇总所有分数。

这样做的优势：

- **Prompt 简单聚焦**：每个 Agent 只需要理解「这一个」指标对应的文本信号
- **输出格式统一**：每个 Agent 输出一个 `[0,1]` 或 `[-1,1]` 范围内的数值 + 置信度
- **可独立优化**：某个指标不准时，只替换对应的 Agent，不影响其他
- **可选择性启用**：用户可以选择只分析感兴趣的子集（如只看 AGI + CB）
- **易于审计**：每个分数的推理链独立可查

### 2. AI Agent 流水线

每个视频被递交给一个 **Video Agent（主 Agent）**，它知道这个视频的「归属社区」（如：这是一个鸣潮视频），然后按以下阶段发起多步骤分析：

```
Video Agent 收到视频 V（归属社区: 鸣潮）
│
├─ 2.1 视频归属判定 & 元信息提取
│
├─ 2.2 社区归属识别（每个 root comment 执行一次）
│   └─ Agent-Community → 输出: inferred_community, confidence
│
├─ 2.3 逐个指标发起分析（每个跨社区评论 × 每个指标各一个 Agent）
│   ├─ Agent-VR  → 串门强度分数 [0,1]
│   ├─ Agent-AGI → 攻击性分数 [0,1] + 细项拆解
│   ├─ Agent-FI  → 友善度分数 [0,1]
│   ├─ Agent-CB  → 拉踩倾向分数 [-1,1]
│   ├─ Agent-DR  → 防御反应分数 [0,1]
│   ├─ Agent-CI  → 冲突卷入分数 [0,1]
│   └─ Agent-EI  → 情绪烈度分数 [0,1] + 情感效价
│
├─ 2.4 量化结果综合（Video Agent 汇总为评论分析向量）
│
└─ 2.5 争议因果分析（跨评论，带群体归属）
```

#### 2.1 视频归属判定 & 元信息提取

- 基于视频标题、描述、标签、UP 主信息，自动判定或确认视频的「归属社区」
- 支持一个视频关联多个社区（如联动/对比类视频）
- 提取视频的核心议题（新版本、角色评测、剧情讨论、强度对比等）

#### 2.2 社区归属识别（Agent-Community）

这是后续所有指标分析的前提。为每条 root comment，由一个 **Agent-Community** 执行：

基于以下信息推断评论者属于哪个玩家群体：

- **评论内容本身**：是否提及特定游戏专有名词、角色名、版本号、社区黑话/梗
- **跨评论者画像**：该用户在本次采集范围内出现在哪些视频下、说了什么（由 Video Agent 汇总）
- **UP 主关联**：基于 troll 已采集的用户数据，辅助判断

输出每个评论者的：

```json
{
  "rpid": 123456,
  "inferred_community": "原神",
  "confidence": 0.85,
  "reasoning": "评论中多次提及'原神'专有名词，未提及本视频所属'鸣潮'相关术语"
}
```

**关键规则**：如果 `confidence < 阈值`（例如 < 0.6），该评论标记为「归属未定」，不纳入跨群体指标计算，仅作为基准行为参考。

#### 2.3 逐个指标发起分析（N 个专指标 Agent）

这是 mirror 最核心的设计。对于**每条跨社区评论**（即 `inferred_community ≠ affiliated_community`），Video Agent 依次或并行发起多个**专指标 Agent（Specialized Metric Agent）**，每个只做一件事、输出一个量化分数：

---

**Agent-VR：串门强度分析**

输入：评论原文 + 评论者出身社区 + 视频归属社区

任务：判断这条评论的「跨社区程度」——这是一次偶然路过还是目的性串门？

输出：

- `vr_score`: [0, 1]  —— 0 = 无明显跨社区意图，1 = 明确的目的性串门
- `confidence`: [0, 1]
- `evidence`: 一句话推理摘要

---

**Agent-AGI：攻击性指数分析**

输入：评论原文 + 评论者出身社区 + 视频归属社区

任务：检测评论中是否存在攻击性、贬低、嘲讽、引战等敌对信号

输出：

- `agi_score`: [0, 1]  —— 0 = 完全无害，1 = 极强攻击性
- `sub_scores`:
  - `hostility`: [0, 1]    敌意
  - `sarcasm`: [0, 1]      讽刺/阴阳怪气
  - `trolling`: [0, 1]     引战意图
- `confidence`: [0, 1]
- `evidence`: 推理摘要

---

**Agent-FI：友善度分析**

输入：评论原文 + 评论者出身社区 + 视频归属社区

任务：检测评论中的友善、友好、善意交流信号

输出：

- `fi_score`: [0, 1]  —— 0 = 无友善信号，1 = 极为友善
- `sub_scores`:
  - `praise`: [0, 1]        赞美
  - `constructive`: [0, 1]   建设性讨论
  - `welcome`: [0, 1]        表示欢迎/感兴趣
- `confidence`: [0, 1]

---

**Agent-CB：拉踩倾向分析**

输入：评论原文 + 评论者出身社区 + 视频归属社区

任务：检测评论中是否存在游戏之间的对比性言论，判断对比的偏向方向

输出：

- `cb_score`: [-1, 1]  —— -1 = 强烈贬低本游戏，0 = 中性/无对比，+1 = 强烈赞赏
- `comparison_targets`: ["原神", "星穹铁道"]
- `confidence`: [0, 1]
- `evidence`: 推理摘要

---

**Agent-DR：防御反应分析**

输入：评论原文 + 评论者出身社区 + 视频归属社区 + 父评论/同级评论（可选）

任务：检测评论是否是对跨社区攻击的防御反击

输出：

- `dr_score`: [0, 1]  —— 0 = 不是防御反应，1 = 明确的防御反击
- `triggered_by`（可选）: 触发源评论的 rpid
- `confidence`: [0, 1]

---

**Agent-CI：冲突卷入度分析**

输入：评论原文 + 评论者出身社区 + 视频归属社区 + 同讨论链上下文

任务：评估评论在争议讨论中的参与深度

输出：

- `ci_score`: [0, 1]  —— 0 = 单次发言即止，1 = 深度卷入
- `turn_count`: 该用户在讨论链中的发言轮次
- `escalation`: [0, 1]  语气是否逐步升级
- `confidence`: [0, 1]

---

**Agent-EI：情绪烈度分析**

输入：评论原文

任务：检测评论的情感强度（不论正面还是负面），辅助校准其他指标的解读

输出：

- `ei_score`: [0, 1]  —— 0 = 极度平淡，1 = 情绪极为强烈
- `valence`: [-1, 1]  —— -1 = 极度负面，0 = 中性，+1 = 极度正面
- `confidence`: [0, 1]

---

#### 2.4 量化结果综合

Video Agent 收集所有专指标 Agent 的输出，汇总为每条评论的**评论分析向量**：

```json
{
  "rpid": 123456,
  "community": {
    "inferred": "原神",
    "confidence": 0.85
  },
  "metrics": {
    "vr": { "score": 0.9,  "confidence": 0.88 },
    "agi": { "score": 0.7,  "confidence": 0.82, "sub_scores": {...} },
    "fi":  { "score": 0.2,  "confidence": 0.75 },
    "cb":  { "score": -0.6, "confidence": 0.90, "comparison_targets": ["原神"] },
    "dr":  { "score": 0.0,  "confidence": 0.95 },
    "ci":  { "score": 0.3,  "confidence": 0.70 },
    "ei":  { "score": 0.8,  "confidence": 0.85, "valence": -0.7 }
  }
}
```

这个向量是后续所有聚合、画像、矩阵生成的基础数据单元。每个分数独立可审计、可追溯。

#### 2.5 争议因果分析（带群体归属）

当两条（或多条）root comment 没有直接的回复关系，但都在争论同一件事，且涉及**不同出身社区**的用户时，**争议因果分析器**由 Video Agent 触发，执行：

1. **主题聚类**：按争议议题分组，区分「强度党 vs 剧情党」「游戏 A vs 游戏 B」「版本内容评价」等话题簇
2. **群体标注**：在每个话题簇中，标记每条评论的出身社区
3. **对立模式识别**：
   - **单边入侵**：全部是社区 B 的人在社区 A 的视频下批评
   - **双边混战**：双方社区成员都有参与互相攻击
   - **内部矛盾外溢**：社区 A 内部的分歧暴露在社区 B 的视频下
4. **因果链推理**：推断争议的触发源头——是哪条评论、哪个观点最先引发了跨群体冲突
5. **冲突升级路径**：识别从低 AGI 到高 AGI 的升级模式，或从友善串门 → 对比拉踩 → 恶意引战的演变

争议分析本身**不依赖**专指标 Agent——它直接分析原文和社区归属。但其输出可以反过来校验各指标的一致性（例如争议链中的评论应该有一致的高 CI 分数）。

### 3. 量化指标

这是 mirror 最重要的设计目标——**每个群体的行为特征必须可量化、可对比**。

#### 3.1 跨群体互动指标

| 指标 | 对应 Agent | 定义 | 聚合计算方式 |
|------|-----------|------|-------------|
| **串门率 (Visit Rate, VR)** | Agent-VR | 群体 A 出现在非本社区视频下的评论占比 | 群体 A 的跨社区评论数 / 群体 A 总评论数 |
| **入侵不对称性 (Invasion Asymmetry, IA)** | 无（由 VR 派生） | A→B 方向与 B→A 方向的串门率比值 | VR(A→B) / VR(B→A) |
| **攻击性指数 (Aggression Index, AGI)** | Agent-AGI | 群体 A 在跨社区发言中恶意言论的平均水平 | avg(agi_score over 群体 A 跨社区评论) |
| **友善指数 (Friendliness Index, FI)** | Agent-FI | 群体 A 在跨社区发言中友善行为的平均水平 | avg(fi_score over 群体 A 跨社区评论) |
| **拉踩倾向 (Comparison Bias, CB)** | Agent-CB | 群体 A 提及其他游戏时的偏好方向 | avg(cb_score over 群体 A 提及对比的评论) |
| **防御反应率 (Defensive Reactivity, DR)** | Agent-DR | 群体 A 成员在本社区视频下对跨社区攻击的反击频率 | avg(dr_score over 群体 A 在本社区视频下的评论) |
| **冲突卷入度 (Conflict Involvement, CI)** | Agent-CI | 群体 A 成员参与跨群体争论的深度 | avg(ci_score over 群体 A 跨社区评论) |
| **情绪烈度 (Emotional Intensity, EI)** | Agent-EI | 群体 A 跨社区发言的平均情绪强度 | avg(ei_score over 群体 A 跨社区评论) |

注意：IA 没有独立 Agent，因为它是由 VR 直接派生的比率，无需 AI 分析。

#### 3.2 社区关系矩阵

所有指标汇总后，生成一个 **n×n 社区关系矩阵**（n = 被分析的游戏社区数）：

```
             目标社区
         原神  崩铁  绝区零  鸣潮  方舟
来源 原神   —    VR   VR     VR    VR
社区 崩铁   VR    —   VR     VR    VR
     绝区零 VR   VR    —     VR    VR
     鸣潮   VR   VR   VR      —    VR
     方舟   VR   VR   VR     VR     —

每个单元格可以展开为多维向量 (VR, AGI, FI, CB...)
对角线（本社区→本社区）为基准行为
```

#### 3.3 可信度指标

| 指标 | 对应 Agent | 定义 |
|------|-----------|------|
| **社区归属置信度 (CAC)** | Agent-Community | AI Agent 对评论者出身社区判断的确信程度 |
| **样本覆盖率 (Sample Coverage, SC)** | 无 | 已分析评论数 / 总评论数 |
| **多模型一致性 (Cross-model Agreement, CMA)** | 全指标 | 抽样校验时不同 LLM 对同一评论同一指标评分的一致性 |
| **跨指标一致性 (Cross-metric Consistency, CMC)** | 全指标 | 同一评论的 AGI 和 FI 是否呈现合理的负相关（异常检测） |
| **人工审计溯源 (Audit Trail)** | 全指标 | 每条指标的分数都附带原始评论引用 + Agent 推理链摘要 |

### 4. 聚合输出

所有视频分析完成后，进行跨视频聚合，生成系统性报告：

#### 4.1 群体行为画像

每个社区最终输出一张行为画像卡片：

```
【原神玩家群体行为画像】
│
├─ 串门活跃度: ★★★★☆  (最爱去: 崩铁 > 鸣潮 > 绝区零)
├─ 攻击性指数: ★★☆☆☆  (跨社区恶意言论: 12%)
├─ 友善指数:   ★★★★☆  (友善串门: 58%)
├─ 最常被贴标签: "OP" "ky" "卫兵"
├─ 关键行为模式:
│  • 在原神视频下内部讨论为主，很少主动提及其他游戏
│  • 在崩铁视频下以友善串门 + 理性讨论为主
│  • 在鸣潮视频下对比拉踩比例显著高于其他目标
│  • 防御反应率高——在本社区受攻击时反击比例很高
└─ 争议热点: "原神 vs 鸣潮 开放世界对比" "剧情淡化争论"
```

#### 4.2 社区关系图谱

基于社区关系矩阵，生成力导向图或弦图：

- **节点**：每个游戏社区
- **边**：串门流量（方向 = A→B，粗细 = VR，颜色 = AGI）
- 视觉上展示：哪些社区之间互动频繁、哪些方向存在攻击性不平衡

#### 4.3 跨社区争议热力图

按「(来源社区, 目标社区, 争议议题)」三维度聚合，展示哪些社区对组合在哪些议题上冲突最激烈。

### 5. 时段对比与趋势分析（可选）

如果启用了分时段分组，mirror 可以追踪跨群体互动行为的**时序变化**：

#### 5.1 时段间对齐

- 同一组社区关系指标在多个时段重复计算
- 关键事件标记（版本更新、新游戏上线、节奏事件）
- 用户群体的跨时段重叠与流失

#### 5.2 趋势指标

| 派生指标 | 定义 |
|----------|------|
| **Δ-AGI** | 攻击性指数随时间的变化量 |
| **关系升温/降温** | 两个社区之间串门率和友善指数的增减 |
| **节奏事件冲击** | 特定事件发生前后攻击性指数的突变 |
| **稳态水平** | 排除事件冲击后的基线攻击性/友善度 |
| **社区生命周期** | 新社区早期 vs 成熟期的跨群体行为演变 |

#### 5.3 输出格式

- **时序数据表**（JSON / CSV）：每行 = 一个时段 × (来源, 目标, 指标)
- **群体行为趋势图**：单个社区的 AGI、FI 等指标随时间变化曲线
- **社区关系演化 GIF/序列图**：社区关系矩阵随时间的演变

## 与 troll 的关系

| 维度 | troll | mirror |
|------|-------|--------|
| 定位 | 数据采集 + 基础统计 | AI 跨群体互动分析 + 量化指标 |
| 数据 | 存原始评论数据 | 存社区归属、行为分类、群体画像、关系矩阵 |
| 输出 | TUI 浏览、CLI 查询 | 社区关系图谱、群体行为画像、跨群体互动矩阵 |
| 依赖 | Bilibili API | troll 数据 + LLM API |
| 扩展性 | 单机 | 单机 + 分布式 Agent |

**路线**：mirror 初期直接 import troll 的 package 复用其数据层和 API 封装层。长远可将公共部分抽取到 `shared/` 或独立为 lib。

## 设计原则

1. **可审计**：每一条 AI 分析结论（社区归属、行为分类）都可以追溯到原始评论 + 推理链，不做「黑箱」判断。
2. **可量化**：避免定性描述，所有群体行为输出标准化数值指标（附带置信度）。
3. **可复现**：相同输入数据 + 相同 Agent 配置 → 相同分析结果。
4. **边界意识**：尊重社区归属判定的不确定性，低置信度结果不纳入统计。
5. **尊重平台**：延续 troll 的限流与多账号设计，不触发 Bilibili 反爬机制。
6. **模块化 Agent**：Comment Agent 可独立测试、独立替换（不同 LLM / 不同 prompt 策略）。

## 快速开始（开发阶段）

```bash
# 构建
cd mirror
go mod tidy
go build -o mirror ./cmd/mirror

# 配置 LLM API Key
export OPENAI_API_KEY=sk-...
# 或
export ANTHROPIC_API_KEY=sk-ant-...

# ── 模式 1: 按社区关键词搜索 + 分析 ──
./mirror analyze -c "原神,星穹铁道,绝区零,鸣潮,明日方舟"

# ── 模式 2: 指定视频列表 ──
# 文件格式：每行一个 BVID，可选 =社区标记
#   BV1xx411c7mD
#   BV1zz456abc=鸣潮
./mirror analyze -f videos.txt

# ── 报告 ──
# JSON 聚合（社区关系矩阵 + 群体行为画像）
./mirror report -d data/analyses/batch-analysis_20260630_190451

# HTML 可视化报告（热力图 + 画像卡片）
./mirror report -d data/analyses/batch-analysis_20260630_190451 -f html -o report.html

# ── 趋势 ──
./mirror trend \
  -d data/analyses/batch-week1_20260601 \
  -d data/analyses/batch-week2_20260608 \
  -o trend.html
```
